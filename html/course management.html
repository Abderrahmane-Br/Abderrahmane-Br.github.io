Distributed course management app

This project is a full-stack app with a distributed database for managing courses

I built this app as the semestrial project for the advanced databases module. It consists of an horizontally sharded database (2 shards) to simulate large database scaling-up and a concurrent hybrid (PHP and Python) crud system. On the front-end it uses Ajax to handle the back-end communication.

For the back-end, Apache is used as a server, the app uses Php to handle all the single threaded operations while Python handles the concurrent operations due to its native threading interface.Finally the database design is implemented in MySQL.
On the other hand, for the front-end, the app is kept simple with vanilla Javascript, Css3 and Html5. In order to give a more fluid experience Ajax is widely used throughout the app.

Challenges

This app is by far the most challenging project I've built, it breaks down to three notable challenges

1st challenge: After developping a fully functional app for a single database and designing a sharded one, the surprise came that the posix library was no longer supported in php thus no concurrent operation was possible. After a lot of searching and on the verge of giving up and rebuilding the app from scratch I stumbeled on the Php-Python cross scripting wich offered a simple yet effective solution.

2nd challenge: The process of sharding and designing the distributed database was one of the most challenging parts of this project due to the abscence of any practical example, I had to look to many different paradigms and approaches that some large companies follow to get this done, I ended up with a similar approach to Oracle's.

3rd challenge: Another problem I encountered was implementing the database normalisation in MySQL since it lacks the support of many normalisation features that one would expect them to be e.g: combined ID of a derived table. It took a lot of tricks and workarounds to get over it.
